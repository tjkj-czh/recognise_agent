# 用地识别智能体 — 服务器安装指南（命令执行版 / ROOT）

> 目标：你本地测试通过后，服务器只需按本指南逐条执行命令，即可完成可部署的企业级运行底座。  
> 说明：本指南不依赖 `.sh` 上传，全部为可复制命令块。

---

## 一、先统一认知（为什么要这样装）

该智能体是“可治理系统”，不是单纯模型调用：

- **系统层**：负责任务编排、状态管理、审计日志、结果回传。
- **能力层（Skills）**：负责分割、识别、投票、VLM裁判、规则校验、统计。
- **数据层**：保存过程与结果，保证可追溯。
- **展示层**：把“经验证结果”可视化给用户。

所以服务器环境必须同时满足：

1. **基础算力与依赖**（GPU + Python + 遥感库）
2. **中间件底座**（PostGIS/Redis/MinIO）
3. **可运维能力**（日志、备份、反向代理、安全）

---

## 二、目标环境与版本（统一基线）

- OS：`Ubuntu 22.04 LTS`
- CUDA：`12.4`
- Python：`3.10`（Conda环境 `landuse`）
- Node.js：`20.x` + `pnpm`
- 数据与中间件：`PostgreSQL 16 + PostGIS 3.4`、`Redis 7`、`MinIO`
- AI推理：`PyTorch cu124`、`vLLM`、`Qwen2-VL-7B-Instruct`

## 六、安装 Docker + Compose

```bash

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(lsb_release -cs) stable" > /etc/apt/sources.list.d/docker.list

apt update
DEBIAN_FRONTEND=noninteractive apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

docker --version
```

## 十二、部署中间件（PostGIS/Redis/MinIO）

```bash
cat > ${BASE_DIR}/docker/docker-compose.yml << 'YAML'
version: "3.8"
services:
  postgres:
    image: postgis/postgis:16-3.4
    container_name: landuse-postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: landuse_agent
      POSTGRES_USER: landuse
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U landuse -d landuse_agent"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: landuse-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    command: redis-server --maxmemory 1gb --maxmemory-policy allkeys-lru
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 5

  minio:
    image: minio/minio:latest
    container_name: landuse-minio
    restart: unless-stopped
    environment:
      MINIO_ROOT_USER: ${MINIO_ACCESS_KEY}
      MINIO_ROOT_PASSWORD: ${MINIO_SECRET_KEY}
    ports:
      - "9000:9000"
      - "9001:9001"
    volumes:
      - miniodata:/data
    command: server /data --console-address ":9001"

volumes:
  pgdata:
  miniodata:
YAML

cd ${BASE_DIR}/docker
set -a; source ${BASE_DIR}/.env; set +a
docker compose up -d
docker compose ps
```

创建 MinIO 桶：

```bash
docker run --rm --network host minio/mc sh -c "\
mc alias set local http://127.0.0.1:9000 landuse PLEASE_CHANGE_STRONG_PASSWORD && \
mc mb -p local/images || true && \
mc mb -p local/tiles || true && \
mc mb -p local/patches || true && \
mc mb -p local/exports || true"
```

> 如果你改了 `MINIO_ACCESS_KEY/MINIO_SECRET_KEY`，上面命令也要同步替换。

## 十七、企业上线前加固（强烈建议）

- **安全**：
  - 立即替换默认密码
  - `.env` 不入库
  - 使用密钥管理（Vault/KMS）
- **运维**：
  - Nginx + HTTPS
  - systemd 托管 API/Worker/VLM
  - Prometheus + 告警（失败率、队列积压、GPU异常）
- **数据保障**：
  - PostgreSQL 每日备份 + 恢复演练
  - MinIO 桶级备份
- **治理要求**：
  - 结果必须保留 `task_id/trace_id/model_version/rule_version`
  - 保证“识别 + 验证 + 统计 + 提示”闭环可追溯

## 十八、常见报错与处理

- **`docker: command not found`**：Docker未安装成功，回到第六章重装。
- **`nvidia-container-cli` 报错**：驱动或 toolkit 不匹配，检查 `nvidia-smi` 和第七章。
- **`rasterio` 安装失败**：走 conda-forge 兜底命令。
- **VLM 显存不足**：降低 `--gpu-memory-utilization`，或先不启VLM。
- **MinIO 桶创建失败**：检查 `MINIO_ACCESS_KEY/SECRET` 与 `mc alias` 参数是否一致。
