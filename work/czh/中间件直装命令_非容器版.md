# 中间件直装命令（非容器版）

> 适用场景：当前环境不使用 Docker / systemd，直接在系统内启动进程。  
> 覆盖组件：PostgreSQL + PostGIS、MinIO。

---

## 0) 统一目录与变量

```bash
export BASE_DIR=/opt/tjkj
mkdir -p ${BASE_DIR}/{postgres,minio}/{data,log}
mkdir -p ${BASE_DIR}/scripts
```

---

## 1) 修复 apt（如之前装 Docker 弄乱依赖）

```bash
dpkg --configure -a || true
apt --fix-broken install -y || true
apt clean
apt update
```

---

## 2) 安装 PostgreSQL 16 + PostGIS 3（固定PG16）

```bash
# 导入 PGDG 仓库（用于安装 PostgreSQL 16）
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor -o /etc/apt/keyrings/postgresql.gpg
chmod a+r /etc/apt/keyrings/postgresql.gpg

echo "deb [signed-by=/etc/apt/keyrings/postgresql.gpg] http://apt.postgresql.org/pub/repos/apt $(. /etc/os-release && echo $VERSION_CODENAME)-pgdg main" \
  > /etc/apt/sources.list.d/pgdg.list

apt update
DEBIAN_FRONTEND=noninteractive apt install -y \
  postgresql-16 postgresql-client-16 postgresql-contrib-16 postgresql-16-postgis-3 \
  wget curl ca-certificates
```

确认二进制路径：

```bash
PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)
echo $PG_BIN
```

## 3) 初始化 PostgreSQL 数据目录（不依赖 systemd）

> 注意：PostgreSQL 不能以 root 身份运行，使用 `postgres` 用户。

```bash
id postgres >/dev/null 2>&1 || useradd -m postgres
chown -R postgres:postgres ${BASE_DIR}/postgres

PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)

su -s /bin/bash postgres -c "$PG_BIN/initdb -D ${BASE_DIR}/postgres/data --encoding=UTF8 --locale=C"
```

配置监听与认证：

```bash
cat >> ${BASE_DIR}/postgres/data/postgresql.conf << 'EOF'
listen_addresses = '*'
port = 5432
max_connections = 200
EOF

cat >> ${BASE_DIR}/postgres/data/pg_hba.conf << 'EOF'
host    all             all             0.0.0.0/0               md5
host    all             all             ::/0                    md5
EOF
```

启动 PostgreSQL：

```bash
PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)
su -s /bin/bash postgres -c "$PG_BIN/pg_ctl -D ${BASE_DIR}/postgres/data -l ${BASE_DIR}/postgres/log/postgres.log start"
```

初始化业务库与扩展：

```bash
# 改成你自己的强密码
export LANDUSE_DB_PASS='ChangeMe_Strong_DB_Password_2026'

PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)

su -s /bin/bash postgres -c "$PG_BIN/psql -v ON_ERROR_STOP=1 <<'SQL'
ALTER USER postgres WITH PASSWORD 'ChangeMe_Postgres_Admin_2026';
DO $$
BEGIN
   IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'landuse') THEN
      CREATE ROLE landuse LOGIN PASSWORD 'ChangeMe_Strong_DB_Password_2026';
   END IF;
END
$$;
CREATE DATABASE landuse_agent OWNER landuse;
\c landuse_agent
CREATE EXTENSION IF NOT EXISTS postgis;
SQL"
```

验证：

```bash
PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)
su -s /bin/bash postgres -c "$PG_BIN/psql -d landuse_agent -c 'SELECT PostGIS_Version();'"
```

## 4) 安装 MinIO（非容器）

下载并安装二进制（固定 2024.06 版本）：

```bash
wget -O /usr/local/bin/minio https://dl.min.io/server/minio/release/linux-amd64/archive/minio.RELEASE.2024-06-13T22-53-53Z
chmod +x /usr/local/bin/minio
/usr/local/bin/minio --version
```

创建 MinIO 运行用户和目录：

```bash
id minio >/dev/null 2>&1 || useradd -r -s /usr/sbin/nologin minio
mkdir -p ${BASE_DIR}/minio/{data,log}
chown -R minio:minio ${BASE_DIR}/minio
```

启动 MinIO：

```bash
# 改成你自己的强密码
export MINIO_ROOT_USER='landuse'
export MINIO_ROOT_PASSWORD='ChangeMe_Strong_Minio_Password_2026'

su -s /bin/bash minio -c "MINIO_ROOT_USER=${MINIO_ROOT_USER} MINIO_ROOT_PASSWORD=${MINIO_ROOT_PASSWORD} \
nohup /usr/local/bin/minio server ${BASE_DIR}/minio/data --address :9000 --console-address :9001 \
> ${BASE_DIR}/minio/log/minio.log 2>&1 &"
```

验证：

```bash
curl http://127.0.0.1:9000/minio/health/live
# 控制台: http://服务器IP:9001
```

可选：安装 `mc` 并创建桶：

```bash
wget -O /usr/local/bin/mc https://dl.min.io/client/mc/release/linux-amd64/mc
chmod +x /usr/local/bin/mc

mc alias set local http://127.0.0.1:9000 ${MINIO_ROOT_USER} ${MINIO_ROOT_PASSWORD}
mc mb -p local/images
mc mb -p local/tiles
mc mb -p local/patches
mc mb -p local/exports
mc ls local
```

## 6) 进程管理（无 systemd 场景）

创建手动启停命令（防止重启后忘记）：

```bash
cat > ${BASE_DIR}/scripts/start_middleware.sh << 'EOF'
#!/usr/bin/env bash
set -e
BASE_DIR=/opt/tjkj
PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)
su -s /bin/bash postgres -c "$PG_BIN/pg_ctl -D ${BASE_DIR}/postgres/data -l ${BASE_DIR}/postgres/log/postgres.log start" || true
su -s /bin/bash minio -c "MINIO_ROOT_USER=landuse MINIO_ROOT_PASSWORD=ChangeMe_Strong_Minio_Password_2026 nohup /usr/local/bin/minio server ${BASE_DIR}/minio/data --address :9000 --console-address :9001 > ${BASE_DIR}/minio/log/minio.log 2>&1 &" || true
EOF
chmod +x ${BASE_DIR}/scripts/start_middleware.sh
```

```bash
cat > ${BASE_DIR}/scripts/stop_middleware.sh << 'EOF'
#!/usr/bin/env bash
set -e
BASE_DIR=/opt/tjkj
PG_BIN=$(ls -d /usr/lib/postgresql/*/bin | sort -V | tail -n1)
su -s /bin/bash postgres -c "$PG_BIN/pg_ctl -D ${BASE_DIR}/postgres/data stop -m fast" || true
pkill -f "/usr/local/bin/minio server" || true
EOF
chmod +x ${BASE_DIR}/scripts/stop_middleware.sh
```

## 7) 常见问题

- `systemctl不可用`：本方案不依赖 systemd，直接 `pg_ctl` + `nohup` 启动。
- `apt broken packages`：先执行第1步修复。
- `PostgreSQL启动失败`：查看 `${BASE_DIR}/postgres/log/postgres.log`。
- `MinIO启动失败`：查看 `${BASE_DIR}/minio/log/minio.log`。
