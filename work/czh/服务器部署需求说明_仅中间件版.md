# 用地识别智能体 — 服务器部署需求说明（仅中间件版）

## 一、目标与范围

### 1.1 目标

当前阶段服务器只做“稳定中间件底座”，包括：

- `PostgreSQL + PostGIS`（业务数据与空间数据）
- `MinIO`（影像与结果文件对象存储）
- （可选）`Redis`（任务队列/缓存，若后续需要）

应用代码、模型调试、前端构建、技能开发均在本地完成。待本地验证通过后，再打包发布到服务器运行。

### 1.2 本阶段不做

- 不在服务器上做完整开发环境（不强制安装完整Python AI依赖）
- 不在服务器上做模型训练/实验
- 不在服务器上做Skill开发
- 不在服务器上做复杂一键安装脚本依赖链

---

## 二、总体部署策略

采用“**本地开发 + 服务器运行中间件 + 发布制品上线**”模式：

1. 服务器先部署并稳定运行中间件（PG/MinIO）
2. 本地完成业务开发、联调、测试、打包
3. 将发布包上传服务器并启动应用
4. 应用只连接服务器中间件，不在服务器重复开发流程

---

## 三、服务器侧需求（必须）

### 3.1 基础环境

- OS：Ubuntu 22.04 LTS（root权限可用）
- Docker + Docker Compose 可用
- 能访问镜像仓库
- 磁盘目录规划：
  - `/root/tjkj/docker`（编排文件）
  - `/root/tjkj/data`（持久化数据）
  - `/root/tjkj/backups`（备份）
  - `/root/tjkj/logs`（日志）

### 3.2 中间件组件

- PostgreSQL 16 + PostGIS 3.4
- MinIO（控制台 + S3 API）
- （可选）Redis 7

### 3.3 持久化要求

- PostgreSQL 数据目录必须挂载卷
- MinIO 数据目录必须挂载卷
- 容器重启后数据不丢失

### 3.4 网络与端口建议

- PostgreSQL：`5432`
- MinIO API：`9000`
- MinIO Console：`9001`
- Redis（可选）：`6379`

> 建议仅内网开放，或通过安全组/IP白名单限制访问。

---

## 四、应用连接契约（本地与服务器统一）

应用发布到服务器后，应通过环境变量连接中间件：

- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `DB_URL`
- `MINIO_ENDPOINT`
- `MINIO_ACCESS_KEY`
- `MINIO_SECRET_KEY`
- `MINIO_BUCKET_IMAGES`
- `MINIO_BUCKET_TILES`
- `MINIO_BUCKET_PATCHES`
- `MINIO_BUCKET_EXPORTS`

可选（若启用异步任务）：

- `REDIS_URL`
- `CELERY_BROKER_URL`
- `CELERY_RESULT_BACKEND`

---

## 五、交付物要求（本阶段）

服务器侧交付：

1. `docker-compose.yml`（仅中间件）
2. `.env`（中间件连接配置，不提交Git）
3. 初始化SQL（如扩展postgis、基础表初始化）
4. MinIO桶初始化说明
5. 健康检查命令清单

本地侧交付：

1. 可运行发布包（后端/前端）
2. 版本号与发布说明
3. 配置模板（`.env.example`）

---

## 六、验收标准（必须通过）

### 6.1 中间件可用性

- PostgreSQL 容器健康检查通过
- PostGIS扩展可查询（`SELECT PostGIS_Version();`）
- MinIO健康接口返回成功
- MinIO桶可创建、可列举

### 6.2 连接可用性

- 应用可成功连接 PostgreSQL
- 应用可读写 MinIO 指定桶
- （若启用）应用可连接 Redis

### 6.3 持久化验证

- 重启容器后数据仍存在
- 备份目录可正常写入

---

## 七、发布流程（简化）

1. 本地开发并测试通过
2. 打包构建产物（后端/前端）
3. 上传服务器（Git tag/压缩包均可）
4. 使用服务器 `.env` 启动应用
5. 执行健康检查与冒烟测试
6. 记录版本与回滚点

---

## 八、风险与约束

- 若服务器仅部署 PG/MinIO，则应用必须避免依赖服务器本地开发工具链
- 凭据必须使用强密码，且不写入仓库
- 生产环境建议启用 TLS、备份、监控告警
- 应用与中间件版本需固定（避免“本地好用、线上不兼容”）

---

## 九、一句话确认

本阶段的正确策略是：**服务器先只部署并稳定中间件（PostgreSQL+PostGIS、MinIO），开发与模型工作全部在本地完成，待验证通过后再将发布包部署到服务器运行。**
