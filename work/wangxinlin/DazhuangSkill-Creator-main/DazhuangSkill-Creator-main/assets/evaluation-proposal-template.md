# 评估前置提案

先用这份提案和用户对齐；在用户没有明确拍板前，不要直接开始正式测评。
这一步是正式评估流程的第一段，不是轻量版评估。用户一旦拍板，后面必须继续走完整个正式流程，直到生成 `review.html` 和 `report.html`。

1. 我初步判断这是什么 skill
- 它主要在解决什么问题
- 它更像哪一类：落地效果型 / 模仿还原型 / 混合型
- 它主要服务的是执行结果、思维方式，还是表达风格
- 我的判断依据是什么
- 我的判断把握有多大

2. 我建议优先按什么方向评
- 我推荐的主评方向是什么
- 我为什么这样建议
- 这个方向最适合看出它有没有价值

3. 除了这个，还可以按哪些方向评
- 可选方向 A：看什么
- 可选方向 B：看什么
- 可选方向 C：看什么
- 哪些方向容易混淆
- 哪些方向这次不建议混在一起

4. 这次可能的主流评估方向
- 方向 1：落地效果
- 方向 2：模仿还原
- 如果是混合型，建议主次权重怎么分

5. 我建议的比较方式
- 单 skill 体检
- 有 skill vs 没 skill
- skill A vs skill B
- skill A vs skill B vs skill C
- 我建议用哪一种，为什么

6. 我建议重点看的维度
- 建议看的 3-5 个主维度
- 每个维度在看什么
- 每个维度为什么重要
- 哪些维度这次明确不看

7. 我建议怎么出测试题
- 应该用什么类型的真实案例
- 为什么这种案例能测出差别
- 需要几类案例
- 哪些案例会造成假象，不建议用

8. 需要你来拍板的事
- 你这次最在意的是哪种“好”
- 你明确不在意什么
- 你要的是单纯好用，还是要像某个人
- 如果要像某个人，你要像的是语气、思维、做事方式，还是结果风格
- 如果是混合型，你希望各自占多大比重
