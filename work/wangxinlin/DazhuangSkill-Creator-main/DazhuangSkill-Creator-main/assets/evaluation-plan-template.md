# 正式评估计划

只有用户已经确认评估标准后，才把内容落成这份正式计划；不要把 AI 自己的推荐版直接当成确认版。
这份计划一旦确认，后面就必须进入正式执行并产出双 HTML；不能降级成聊天结论、Markdown 结论或普通 review。

1. 本次要评什么
- 评估对象是谁
- 是单 skill、无 skill 对照，还是多 skill 对比

2. 本次最终采用的评估方向
- 主方向
- 次方向
- 各自权重

3. 本次重点看哪些维度
- 维度 1
- 维度 2
- 维度 3
- 维度 4（可选）
- 维度 5（可选）

4. 本次明确不看什么
- 不纳入主评的项目
- 只作观察、不作主判的项目

5. 本次测试案例怎么选
- 真实案例来源
- 样本类型
- 样本数量
- 为什么这些样本能区分好坏

6. 本次如何比较
- 同模型
- 同环境
- 同预算
- 同时间约束
- 是否盲评

7. 本次输出怎么写
- 给出总判断
- 给出分维度判断
- 给出证据
- 给出适用场景和不适用场景
- 哪些维度没提升
- 哪些维度反而退步
- `review.html`：基础证据工作台
- `report.html`：给用户看的完整报告
- 两个 HTML 必须一起生成
