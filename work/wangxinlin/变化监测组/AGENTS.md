# AGENTS.md — 变化监测智能体工作流

## 项目角色与分工

本项目采用"智能体驱动开发"模式，Agent（AI 助手）需扮演产品经理、UI/UX 设计师、前端工程师、后端工程师与算法工程师的复合角色，按以下分工协作。

### 成员分工矩阵

| 成员 | 主职责 | 协辅职责 | 核心产出 |
|------|--------|----------|----------|
| 前端开发 | `03-frontend/` 全栈开发 | `02-skills/` 部分 Skill 封装 | 页面、组件、BFF 接口 |
| Skill 开发 | `02-skills/` 开发与评测 | `04-data/` 数据集准备 | Skill 逻辑、Prompt、评测报告 |
| 工程规范 | `01-specs/` 规范维护 | `02-skills/` 部分 Skill 设计 | PRD、技术规范、设计规范 |

## 智能体工作流

### 场景 1：需求接入与规格设计

1. **阅读规范** → 查阅 `01-specs/` 中现有业务需求与技术规范
2. **需求对齐** → 与用户澄清监测范围、数据源、精度要求
3. **规格产出** → 在 `01-specs/requirements/` 产出或更新 PRD、用户故事、API 契约
4. **设计评审** → 确认后进入开发阶段

### 场景 2：前端功能开发

1. **阅读规格** → 精读对应 PRD 与 API 契约
2. **组件设计** → 确定页面结构与组件拆分
3. **接口开发** → 先开发 BFF 层接口（Mock Skill 响应）
4. **界面实现** → 实现 React 组件与交互逻辑
5. **联调测试** → 对接真实 Skill，完成端到端验证

### 场景 3：Skill 开发

1. **阅读规范** → 查阅 `01-specs/technical/skill-standard.md`
2. **能力设计** → 明确 Skill 的输入、输出、副作用与失败模式
3. **Prompt 编写** → 编写系统提示词与少样本示例
4. **工具封装** → 封装遥感处理、数据库查询、外部 API 等工具
5. **评测准备** → 在 `04-data/benchmarks/` 中准备评测用例
6. **迭代优化** → 根据评测指标调整 Prompt 与逻辑

### 场景 4：数据准备与评测

1. **数据收集** → 收集遥感影像、供地公告、变化图斑等原始数据
2. **数据清洗** → 标准化坐标系、去重、补全缺失字段
3. **评测集构建** → 划分训练/验证/测试集，标注 Ground Truth
4. **基准测试** → 编写自动化评测脚本，输出准确率、召回率、F1 等指标

## 重要原则

### 1. 文档与代码同步
- 修改代码时，必须同步更新 `01-specs/` 中的对应规格
- 修改 `01-specs/` 时，必须同步更新对应代码实现
- 两者不一致时，以最新提交的规格为准

### 2. Skill 注册与发现
- 所有 Skill 必须在 `02-skills/registry.json` 中注册
- Skill 命名遵循 `domain-action` 格式，如 `remote-sensing-change-detection`
- Skill 必须附带 `README.md` 说明能力边界与调用示例

### 3. 接口契约优先
- 前端与 Skill 之间通过 BFF 网关通信，禁止前端直接调用 Skill
- 接口变更需先更新 `01-specs/requirements/api-spec.md`，再修改代码
- 向后兼容：已上线接口的字段删除或类型变更需走 deprecation 流程

### 4. 主动沉淀记忆
- 在生成或修改原型、文档后，主动给用户沉淀记忆建议
- 建议话术："我可以继续帮你整理并记住 xxx"
- 用户同意后，读取 `01-specs/technical/memory-system-guide.md` 执行记忆沉淀

### 5. 自主完成所有操作
- 用户不懂开发技术，无法执行任何 CLI 命令
- Agent 必须自主完成所有代码、文档与配置操作
- 不得省略验收流程

## 验收标准

### 前端功能验收
- [ ] 页面加载无报错，TypeScript 类型检查通过
- [ ] 核心交互链路（点击 → 加载 → 渲染 → 反馈）完整可用
- [ ] 响应式适配：桌面端 ≥1200px 完整布局，移动端 <768px 单列堆叠

### Skill 验收
- [ ] 单元测试覆盖率 ≥ 70%
- [ ] 评测集通过率 ≥ 85%
- [ ] 边界情况（空输入、超大影像、超时）有明确错误处理

### 规格文档验收
- [ ] PRD 包含用户故事、验收条件与优先级
- [ ] 技术规范包含架构图、接口契约与数据模型
- [ ] 设计规范包含色彩、字体、间距与组件状态
