## 2. 技术选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | React 19 + TypeScript + Tailwind CSS | 组件化开发，响应式适配 |
| 状态管理 | Zustand | 轻量，支持异步与持久化 |
| 地图 | Leaflet / MapLibre GL | 2D 卫星底图 + 矢量叠加 |
| BFF | Node.js + Fastify | 高性能，低开销 |
| 通信协议 | tRPC / REST OpenAPI | 类型安全的前后端通信 |
| Skill 运行时 | Python + LangChain | LLM 编排与工具调用 |
| 数据库 | PostgreSQL + PostGIS | 空间数据支持 |
| 缓存 | Redis | 会话、热点数据缓存 |
| 容器化 | Docker + Docker Compose | 开发环境一致性 |

## 3. 数据流

### 3.1 监测指令流
```
用户输入 → 前端 → BFF /skill/orchestrate → Agent 编排器 → 解析意图
    → 调用 remote-sensing-change-detection Skill
    → 返回变化图斑 + 置信度 + 对比影像
    → BFF 组装响应 → 前端渲染卷帘对比与图斑列表
```

### 3.2 报告生成流
```
用户点击"生成报告" → 前端 → BFF /report/generate
    → Agent 编排器调用 report-generation Skill
    → Skill 查询数据库 + 读取模板 → 组装 Markdown / HTML
    → BFF 转 PDF/Word → 返回下载链接
```

## 4. 模块边界

### 前端（03-frontend）
- **职责**：用户界面渲染、交互状态管理、地图可视化
- **禁止**：直接调用遥感处理库、直接访问数据库

### BFF（03-frontend/backend-for-frontend）
- **职责**：前端请求聚合、鉴权、参数校验、Skill 调用网关
- **禁止**：业务逻辑下沉（变化检测算法、Prompt 工程）

### Skill（02-skills）
- **职责**：AI 能力封装（意图理解、变化检测、报告生成）
- **禁止**：直接操作前端 DOM、暴露未鉴权的 HTTP 端点
