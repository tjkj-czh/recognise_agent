# 用地识别智能体 — 服务器软件安装指南（可直接部署版）

> 适用目标：你在本地测试完成后，把代码部署到 Ubuntu 22.04 GPU 服务器即可运行。

## 一、适用环境与目标

- **操作系统**：Ubuntu 22.04 LTS
- **GPU/CUDA**：NVIDIA GPU + CUDA 12.4
- **部署方式**：Docker Compose（开发/测试）+ 预留生产升级能力
- **应用技术栈**：
  - 后端：FastAPI + Celery
  - 数据：PostgreSQL(PostGIS) + Redis + MinIO
  - 模型：PyTorch + vLLM（Qwen2-VL）
  - 前端：Node.js 20 + pnpm（用于构建产物）

---

## 二、你需要在服务器完成的配置（Checklist）

### 1) 基础与系统

- 更新系统包
- 安装基础工具（git/curl/wget/build-essential 等）
- 安装地理依赖（gdal/proj）

### 2) 容器与GPU

- 安装 Docker + Compose 插件
- 配置 NVIDIA Container Toolkit
- 验证容器内 `nvidia-smi`

### 3) Python 运行环境

- 安装 Miniconda
- 创建 `landuse` Python 3.10 环境
- 安装核心依赖（torch/FastAPI/Celery/GDAL生态/vLLM）

### 4) 前端构建工具

- 安装 Node.js 20
- 安装 pnpm

### 5) 目录与配置

- 创建统一部署目录：`~/tjkj/{docker,logs,models,data,backups}`
- 生成 `.env`（密码改为强口令）
- 生成 `docker-compose.yml`（PostGIS/Redis/MinIO）

### 6) 可选但建议

- 安装 Nginx 作为反向代理
- 放通必要端口（如你启用 UFW）
- 启用 systemd 托管应用进程

---

## 三、一键脚本说明

已为你生成脚本：`work/setup_server_oneclick.sh`

### 脚本能力

- 自动安装：基础依赖、Docker、NVIDIA Toolkit、Miniconda、Python 环境、Node/pnpm
- 自动生成：
  - `~/tjkj/.env`
  - `~/tjkj/docker/docker-compose.yml`
- 自动创建：MinIO 需要的桶（`images/tiles/patches/exports`）
- 默认不启动你的业务代码（只把底层环境铺好）

### 使用方式

```bash
cd ~/tjkj/work
chmod +x setup_server_oneclick.sh
./setup_server_oneclick.sh
```

### 关键环境变量（可选）

你可以执行前覆盖默认值：

```bash
export LANDUSE_DB_PASSWORD='改成强密码'
export LANDUSE_MINIO_USER='landuse'
export LANDUSE_MINIO_PASSWORD='改成强密码'
export LANDUSE_BASE_DIR="$HOME/tjkj"
./setup_server_oneclick.sh
```

## 四、脚本执行后你要做的两件事

1. **把本地测试通过的代码上传到服务器**（建议 Git + tag）。
2. **按你的项目启动命令启动后端/worker/前端产物**（脚本已把系统环境准备好）。

## 五、验收命令（建议逐条执行）

```bash
# Docker
Docker --version 2>/dev/null || docker --version
docker compose version

# GPU in Docker
docker run --rm --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi

# Python env
source "$HOME/miniconda3/etc/profile.d/conda.sh"
conda activate landuse
python --version
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"

# Middleware
cd "$HOME/tjkj/docker"
set -a; source "$HOME/tjkj/.env"; set +a
docker compose ps

# MinIO buckets（若已创建）
docker exec landuse-minio mc ls local || true
```

## 六、生产化建议（上线前）

- 所有密码从 `.env` 迁移到密钥管理（如 Vault/KMS）
- Nginx 强制 HTTPS
- 数据库和对象存储开启定时备份
- 加 Prometheus + 告警
- 业务进程使用 `systemd` 或容器编排托管
