# 测评规范

## 目标
验证该 skill 在两类能力上的稳定性：
- **触发能力**：用户提到 TIFF 上传、切片、分割、绘制联调时，应触发此 skill。
- **执行能力**：给定可用服务与样例影像时，应跑通主链路并给出结构化结果。

## 测评项
1. **结构校验**：`quick_validate.py --strict` 必须通过。
2. **触发测评**：使用 `evals/trigger-eval-set.json`，要求准确率 = 0.8。
3. **链路烟测（可选）**：提供真实 tiff 后跑完整链路，要求：
   - 有 `image_id`
   - 有 `task_id`
   - 任务终态为 `success/failed` 之一
   - 能拉取到 `vectors.geojson` 或输出明确失败原因

## 输出要求
测评输出至少包含：
- 执行命令
- 通过/失败统计
- 失败样本（query 或步骤）
- 建议修复方向
