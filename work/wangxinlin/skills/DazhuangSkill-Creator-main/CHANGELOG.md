# 更新日志

## v1.5.11 - 2026-04-11

- 现在正式评估不会只给你一个 review 了，而是会硬性产出两份 HTML。
- 第一份还是老朋友 `review.html`，继续当基础证据工作台，不改它这条线。
- 第二份是新的 `report.html`，专门给人看：会把这次的测评计划、维度、案例、提示词、各对象回答、评分和结论都摊开。
- 还加了一个统一脚本 `generate_eval_artifacts.py`，一次跑完双产物；少一个文件，都不算这次正式评估做完。
- 我又把旁路也补死了：就算有人直接单独跑 `generate_review.py` 或 `generate_report.py`，系统也会默认自动补齐另一份；补不齐就直接失败，不让单份 HTML 假装“评估已完成”。

## v1.5.9 - 2026-04-11

- 修了一个大 bug：以前有人一说“帮我测评这个 skill”，系统还是可能直接开测。现在不行了，第一反应必须先把测评方案拿出来跟人对齐。
- 也就是说，以后不能跳过“你到底想按什么标准评”这一步，更不能自己脑补一把尺子就开始打分。
- 我还补了回归测试，专门盯住这条规则，防止以后又偷偷滑回“直接测评”。

## v1.5.8 - 2026-04-11

- 把评估这条路又补完整了一截：现在如果你拿 3 个同类 skill 一起比，benchmark 不会再只盯着前两个看了，第三个也会老老实实显示出来。
- benchmark 变得更较真了：如果题目没写自己在测哪条维度，或者写了计划里根本没有的维度，或者计划里的关键维度根本没人测到，它会直接拦住。
- review 现在会顺手把“最后结论必须包含什么”也带出来，这样你看结果时不容易又把前面说好的标准忘掉。
- 另外补了一份“前置对齐怎么开口聊”的话术骨架，想少想一点的人可以直接照着开场。

## v1.5.7 - 2026-04-11

- 以前评估有点像一上来就考试，现在会先停一下，先说清楚“这次到底比什么”。
- 新增了两张固定小单子：`评估前置提案` 和 `正式评估计划`。先对齐尺子，再开始量长短。
- benchmark 和 review 现在默认更严格了：如果还没有 `eval-plan.json`，会先拦住你，不让糊里糊涂往下跑。
- 如果你只是想兼容老结果，也留了后门：可以自己明确加 `--allow-missing-eval-plan`。
- README、评估文档、模板、viewer 和回归测试都一起补齐了，这样这套新流程不只是“说说而已”。

## v1.5.6 - 2026-04-11

- 把一个“容易误会”的点讲清楚了：当记忆模式最后是 `lessons` 或 `adaptive` 时，会自动补上 `references/` 和 `scripts/`，不是乱改你的参数。
- 终端参数说明和文档都补了这条规则，减少“我明明没写 scripts 为什么出来了”的困惑。
- 修了更新检查里的一个显示问题：节流模式下，不会再把比本地更老的缓存版本当成 `latest` 展示。
- 返回结果里新增了 `cached_remote_version`，方便你区分“这次联网拿到的最新版本”和“本地缓存看到的远端版本”。

## v1.5.5 - 2026-04-11

- 修了一个真 bug：`quick_validate.py` 现在会忽略代码块里的 `# 标题`，不再把模板代码误判成顶级 section。
- 选了 `--resources assets` 后，会默认创建 `assets/output-format.md`，不再和 `--examples` 绑在一起。
- 初始化时如果检测到“报告/评审/表格”这类固定格式信号，但你还在用内联 `# 输出格式`，会提前给出迁移到 `assets/output-format.md` 的强提醒。
- 补了 3 个回归测试，防止这几处卡点再次出现。

## v1.5.4 - 2026-04-11

- 修了一个“指路牌指到空气”的问题：没加 `--examples` 时，不会再让模板去读不存在的 `references/examples.md`。
- `quick_validate.py` 新增了 `--strict`。这个模式会拦住 `TODO`、`TBD` 和占位词，避免半成品被当成成品。
- 打包前现在默认走严格校验，不会把“还没填完”的 skill 直接打包出去。
- 记忆层文案也更清楚了：如果你已经手动选了模式，就不会再写成“还要再判一次”。
- 补了回归测试，防止这些坑以后偷偷回来。

## v1.5.3 - 2026-04-11

- 修了一个“第一次就容易卡住”的小坑：如果你忘了写 `--path`，现在会直接告诉你两种补救办法，还给你一条能复制的命令。
- 新建时会告诉你“这个路径是从哪里来的”（命令行还是 `config.yaml`），不再让人猜。
- `quick_validate.py` 的提示更直白了：如果 Step 4 少了 `retry` 或 `failure` 事件命令，会直接给可粘贴的那一行。
- 文档也同步补了“最容易踩坑点”和 Step 4 的补全示例，第一次用更顺手。
- 补了回归测试，防止以后又把 `--path` 的引导写丢。

## v1.5.2 - 2026-04-11

- 新建 skill 时，“要不要开记忆”现在是必做检查，不能悄悄跳过。
- 如果你用 `memory_mode=auto`，但没写 `--intent`，而且系统先判断成 `off`，会先停下来提醒你补充信息。
- 新建流程里会更清楚地打印“记忆判型结果”，让你一眼看到这次是怎么判的。
- 文档也补上了这条硬要求：就算最后选 `off`，也要先判断并说清楚理由。
- 补了回归测试，防止以后又出现“没想清楚就把记忆关掉”。

## v1.5.1 - 2026-04-11

- 修了一个“容易误关记忆”的坑。
- 现在只要最后是 `memory_mode=off`（命令行或配置文件），但系统判断这个 skill 明明更该开记忆，就会先拦住并提醒你。
- 只有你明确加 `--force-memory-off`，才会真的强制关闭记忆。
- 补了回归测试，防止这个坑以后再出现。

## v1.5.0 - 2026-04-11

- 这是一次大更新：skill 现在更会“长记性”了。
- 新建 skill 时，系统会先判断记忆模式；你也可以手动选。
- 记忆模式有 4 种：`off`、`adaptive`、`lessons`、`auto`（默认）。
- 开记忆后，会有 3 个“小本本”：状态本（`memory-state.json`）、过程本（`memory-events.jsonl`）、经验本（`memory-lessons.md`）。
- 同一个坑反复出现，会先写进经验本；后面一直做对，这条经验会升级成 `SKILL.md` 里的硬规则。
- 新增防回归测试，避免“已经退休的经验”马上被重新激活。
- `quick_validate.py` 更严格了：记忆模式少关键步骤会直接提醒。
