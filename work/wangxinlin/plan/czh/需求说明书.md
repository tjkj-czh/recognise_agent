## 一、文档目标与统一基线

本版本用于统一原《需求说明书》《技术实施方案》《软件安装指南》中的冲突项，并形成可落地企业级基线。

### 1.1 统一技术基线

- 操作系统：`Ubuntu 22.04 LTS`
- GPU与驱动：`CUDA 12.4`（NVIDIA 驱动按官方兼容矩阵安装）
- Python：`3.10.x`（统一采用 Conda 环境）
- 前端：`Vue 3 + Vite + Element Plus + Cesium`
- 后端：`FastAPI + Celery + Redis`
- 数据：`PostgreSQL 16 + PostGIS 3.4 + MinIO`
- 模型：`SAM / DeepLabV3+ / UNet / Qwen2-VL（vLLM）`

### 1.2 统一设计原则

- 输出可信：仅输出“已验证”结果（投票 + VLM + 规则校验）
- 安全可控：凭据不硬编码、最小权限、全链路审计
- 可运维：可观测、可回滚、可降级
- 可扩展：模型与规则配置化，避免代码硬耦合

## 二、业务目标与范围

### 2.1 业务目标

- 支持遥感影像上传后自动完成图斑分割、地类识别、验证与统计。
- 输出可解释、可追溯、可审计的结构化结果，支撑业务研判。
- 对重点规则触发场景生成“参考性提示”，不输出行政定性结论。

### 2.2 范围定义

- **In Scope（v1.0）**
  - GeoTIFF 上传与元信息校验
  - 图斑分割与标准化
  - 多模型识别、投票验证、VLM 裁判、规则后校验
  - 统计汇总、合规提示
  - Cesium 可视化与结果筛选、报表导出
  - 全链路日志、任务状态追踪
- **Out of Scope（v1.0）**
  - 多租户计费系统
  - 在线模型训练平台
  - 跨区域多活容灾（可在 v2.0 规划）

## 三、用户与权限

### 3.1 用户角色

- 一线作业员：上传影像、查看识别与统计结果
- 质检人员：查看验证链路、定位存疑图斑
- 项目负责人：查看汇总报表、风险提示、任务进度
- 平台运维：部署、监控、告警、备份恢复

### 3.2 权限模型（RBAC）

- `viewer`：只读结果与报表
- `operator`：上传与发起任务
- `reviewer`：查看验证细节与标注存疑
- `admin`：系统配置、规则版本、模型开关

## 四、核心业务流程

```text
上传GeoTIFF
  -> 影像校验与元信息提取
  -> 预处理与切片
  -> 图斑分割与清洗
  -> 多模型识别
  -> 投票验证
  -> 投票争议进入VLM裁判
  -> 规则后校验
  -> 分类聚合与统计
  -> 合规提示生成
  -> 可视化展示与报表导出
```

## 五、功能需求（FR）

### 5.1 影像上传与校验

- FR-01：支持 GeoTIFF 上传（单文件上限 2GB，支持拖拽与按钮上传）
- FR-02：元信息自动提取（坐标系、分辨率、波段、范围）
- FR-03：格式与空间参考校验（缺失或非法需明确报错）
- FR-04：非目标坐标系时提供“可配置转换策略”

### 5.2 分割与标准化

- FR-10：预处理与切片（默认 256×256，步长 128，可配置）
- FR-11：分割输出 Polygon/MultiPolygon
- FR-12：碎片合并与边界修复
- FR-13：图斑标准化输出（ID、几何、面积、质心、来源索引）

### 5.3 识别与验证

- FR-20：并行多模型识别并保留各模型原始结果
- FR-21：投票机制支持 `unanimous/majority/disputed`
- FR-22：争议图斑触发 VLM 裁判，输出标签与理由
- FR-23：规则后校验（空间合理性/邻接合理性/面积合理性）
- FR-24：标签映射到业务标准分类，未映射标签归“其他”并留痕

### 5.4 统计与提示

- FR-30：图斑级聚合（地类面积、占比、混合判定）
- FR-31：区域级统计（按一级/二级类）
- FR-32：合规提示输出“自动提示/关注提示”两级
- FR-33：提示口径必须为参考性建议，禁止行政定性措辞

### 5.5 交互与可视化

- FR-40：上对话区 + 下 Cesium 地球布局
- FR-41：地类颜色、边界样式、状态可视化（投票一致/VLM裁断/存疑）
- FR-42：图斑详情弹窗（主导类别、占比、验证理由、提示）
- FR-43：筛选能力（地类、验证状态、风险类型）
- FR-44：报表导出（JSON/CSV）

### 5.6 日志与审计

- FR-50：记录每个 Agent/Skill 的输入摘要、输出摘要、耗时、状态
- FR-51：任务级全链路追踪（`trace_id/task_id`）
- FR-52：失败任务可回放与重试（幂等）

## 六、非功能需求（NFR）

### 6.1 性能指标（P95）

- 上传吞吐：≥10MB/s（内网）
- 1GB 影像全链路处理：≤15 分钟
- 单图斑识别：≤2 秒
- 区域统计：≤3 秒
- 前端渲染：图斑 ≤500 时无明显卡顿

### 6.2 可用性与可靠性

- 全链路成功率：≥95%
- API 可用性：月度 ≥99.5%
- 任务状态持久化：支持中断恢复
- 关键任务失败：支持自动重试与人工重放

### 6.3 安全与合规

- 凭据管理：禁止硬编码，统一环境变量或密钥管理
- 访问控制：RBAC + 审计日志
- 传输安全：生产启用 TLS（HTTPS）
- 数据隔离：任务级数据隔离与最小权限访问

### 6.4 可观测性

- 指标：Prometheus（任务耗时、失败率、队列深度）
- 日志：结构化日志（JSON）
- 链路：OpenTelemetry（可选）
- 告警：任务积压、失败率突增、模型服务不可用

## 七、数据契约（统一输出结构）

输出继续沿用 v1.2 的核心结构，但新增以下强制字段：

- `trace_id`：全链路追踪ID
- `model_version`：模型版本信息（含哈希或tag）
- `rule_version`：规则版本
- `data_version`：分类体系版本
- `created_at/updated_at`：时间戳（ISO 8601）

统一响应包络：

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "trace_id": "..."
}
```

## 八、交付与验收标准

### 8.1 阶段验收

- 阶段A（底座可用）：上传、入库、预览闭环通过
- 阶段B（识别可用）：分割、投票、映射、统计通过
- 阶段C（验证可用）：VLM裁判、规则校验、存疑闭环通过
- 阶段D（企业可运维）：监控告警、审计日志、备份恢复演练通过

### 8.2 v1.0 验收门槛

- 标准样本全链路通过率 ≥95%
- 最终识别准确率（经验证后） ≥93%
- 面积占比计算准确率 ≥99%
- 合规提示触发准确率 ≥95%
- Sev-1 缺陷 = 0，Sev-2 缺陷可控且有缓解方案

## 九、依赖与前置条件

- 影像数据满足 GeoTIFF 与空间参考要求
- DEM 等辅助数据（若启用高程规则）需提前准备
- 模型权重来源、版本与校验值（checksum）须登记在模型清单

---

## 十、风险与降级策略

- GPU资源不足：分时加载模型、限制并发、启用轻量模型
- VLM不可用：仅保留投票+规则校验，争议标记“存疑待复核”
- PostGIS压力高：空间索引、分区与范围查询约束
- 前端大图斑卡顿：切换 Primitive 渲染与分级加载

---

## 十一、术语

- 图斑：分割后空间同质区域
- 已验证：通过投票/VLM/规则校验后的可信结果
- 存疑：未形成可信结论，需人工复核
- 合规提示：参考性业务提示，不构成行政结论
