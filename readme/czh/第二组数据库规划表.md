# 第二组数据库规划表

## 一、总体数据流

```
原始影像 ──→ [张翔：图斑分割质检] ──→ [程志慧：图斑地类判别] ──→ [王新林：用地合规提示] ──→ 业务输出
                │                           │                            │
                ├─ 标准化图斑                ├─ 图斑地类结果              ├─ 合规提示
                ├─ 质量标签                  ├─ 置信度                    ├─ 风险提示
                └─ 低质量区域标记            ├─ 混合地类提示              ├─ 复核建议
                                            └─ 低置信区标记              └─ 判读摘要
```

---

## 三、表关系总览

```
t_image ──1:N──→ t_patch ──1:1──→ t_patch_classification ──1:N──→ t_compliance_prompt
                   │                      │                               │
                   │                      ├──1:N──→ t_patch_class_breakdown │
                   │                      │                               │
                   │                      ├──1:N──→ t_low_confidence_area  │
                   │                      │                               │
                   │                      └──1:1──→ t_patch_summary ←──────┘
                   │
                   └──────→ t_eval_sample（关联图斑）

t_classification_system ──1:N──→ t_label_mapping
        │
        ├──1:N──→ t_patch_class_breakdown
        │
        ├──1:N──→ t_confidence_threshold
        │
        └──1:N──→ t_compliance_rule

t_compliance_rule ──1:N──→ t_compliance_prompt

t_agent_log（独立，记录所有Agent执行过程）
t_error_registry（独立，记录所有已知问题）
```

## 五、关键字段约定

| 约定项 | 说明 |
|--------|------|
| 主键策略 | 业务表用VARCHAR自然主键，日志/明细表用SERIAL自增 |
| 坐标系 | 统一使用CGCS2000，所有空间字段保持一致 |
| 时间字段 | 一律使用TIMESTAMP，存UTC时间 |
| JSON字段 | quality_issues、input_data、expected_output等使用JSONB类型 |
| 状态字段 | 统一使用VARCHAR而非ENUM，便于扩展 |
| 删除策略 | 不做物理删除，通过status字段做软删除 |
| 版本字段 | 所有配置类表（分类体系、标签映射、规则、阈值）均带version字段，支持版本管理 |

## 六、数据流转示意

以一个图斑从输入到最终输出为例：

```
1. 张翔写入：t_image → t_patch（标准化图斑 + 质量标签）

2. 程志慧读取：t_patch
   程志慧写入：
     t_patch_classification（主导地类 + 置信度 + 混合标记 + 难判标记）
     t_patch_class_breakdown（各类面积占比 + 置信度）
     t_low_confidence_area（低置信区位置 + 原因 + 建议）

3. 王新林读取：t_patch + t_patch_classification + t_patch_class_breakdown + t_low_confidence_area
   王新林写入：
     t_compliance_prompt（合规提示 + 风险类型 + 复核建议）
     t_patch_summary（图斑判读摘要）
```
